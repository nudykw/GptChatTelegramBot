# Опис проєктів

🇺🇸 [English](projects.md) | 🇺🇦 Українська

Цей документ надає огляд кожного проєкту у рішенні **GPTChatTelegramBot**.

## [TelegramBotApp](../TelegramBotApp)
**Головний хост застосунку**

Це точка входу у застосунок. Відповідає за:
- **Ініціалізацію бота**: Налаштування `ITelegramBotClient` та конфігурацію полінгу для отримання повідомлень.
- **Впровадження залежностей**: Організацію реєстрації та використання сервісів із `ServiceLayer` і `DataBaseLayer`.
- **Управління конфігурацією**: Зчитування та застосування налаштувань з `appsettings.json`.

---

## [ServiceLayer](../ServiceLayer)
**Серце бізнес-логіки**

Тут відбувається все, що пов'язано з обробкою запитів користувачів. Ключові обов'язки:
- **Інтеграції з AI**: Оркестрація викликів до OpenAI, Gemini та інших провайдерів через `IChatService`.
- **Фабричний патерн**: Централізоване створення сервісів через `IChatServiceFactory` на основі динамічної конфігурації.
- **Відмовостійкість**: Автоматичне перемикання провайдерів через обгортку `ResilientChatService`.
- **Логіка білінгу**: Розрахунок вартості використання AI (токени) та підготовка білінгових записів для бази даних.
- **Обробка аудіо**: Транскрипція голосових повідомлень через Whisper/OpenAI.
- **Абстракції сервісів**: Визначення `IChatService` та інших інтерфейсів для підтримки модульності.

### [Конфігурація сервісів](../TelegramBotApp/appsettings.sample.json)

Бот підтримує кілька AI-провайдерів. Налаштовуються у `appsettings.json`:

```json
"AppSettings": {
  "ChatProviders": [
    {
      "Name": "OpenAI",
      "ProviderType": "OpenAI",
      "ApiKey": "sk-...",
      "ModelName": "gpt-4o-mini"
    },
    {
      "Name": "Gemini",
      "ProviderType": "Gemini",
      "ApiKey": "AIza...",
      "ModelName": "gemini-pro"
    }
  ]
}
```

> [!TIP]
> Провайдери з ключами-заглушками на кшталт `[YOUR_GEMINI_API_KEY]` автоматично ігноруються `ChatServiceFactory`, тому їх можна залишати в конфігурації до отримання реальних ключів.

## [DataBaseLayer](../DataBaseLayer)
**Шар збереження даних**

Керує базою даних та всіма операціями з нею.
- **Моделі**: Визначає структуру даних для `TelegramUserInfo`, `HistoryMessage` і `GptBilingItem`.
- **Міграції**: Містить історію міграцій EF Core для підтримки актуальності схеми бази даних.
- **Контекст**: Клас `StoreContext` — основний інтерфейс для операцій з базою даних.
- **Мультипровайдерність**: Підтримує SQLite, PostgreSQL, MySQL та SQL Server через `ProviderSpecificMigrationsAssembly`.

## [GeminiChat.DotNet](../ServiceLayer/Services/GeminiChat.DotNet)
**Внутрішній клієнт Gemini**

Спеціалізований проєкт у структурі `ServiceLayer`, що зосереджується виключно на Google Gemini API.
- Надає чисту обгортку HTTP-клієнта для Gemini.
- Обробляє специфічні формати JSON-запитів/відповідей, що вимагає Google.
